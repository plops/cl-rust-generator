# Architecture & Refactoring Master Plan: `cl-rust-generator`


The objective is to transform the monolithic, 1100-line `rs.lisp` into a modular, maintainable AST-to-Rust transpiler utilizing a registry-based dispatch system. We will eschew fragile internal parenthesis-management logic in favor of delegating AST formatting to `rustfmt`, and we will externalize examples to keep the core compiler tree clean.

Follow these phases sequentially. Do not proceed to the next phase until the checkpoint verification is successful.

## Phase 4: The Registry Architecture

We will eliminate the giant `case` statement, replacing it with a hash-table dispatch registry. This adheres to the Open/Closed Principle.

**Task 4.1: Implement the Registry Core**
In `registry.lisp`, add the macro and dynamic variables:

```lisp
(in-package :cl-rust-generator)

(defvar *emit-registry* (make-hash-table :test 'eq)
  "Maps AST node symbols to their emission functions.")

;; Dynamic variables to replace explicit parameter passing
(defvar *emit-level* 0)
(defvar *hook-defun* nil)

(defmacro def-emitter (node-name (args-var) &body body)
  "Registers an AST node emitter."
  `(setf (gethash ',node-name *emit-registry*)
         (lambda (,args-var)
           (flet ((emit (code &optional (dl 0))
                    (let ((*emit-level* (+ *emit-level* dl)))
                      (emit-rs :code code))))
             ,@body))))
```

**Task 4.2: Update `emit-rs` in `core.lisp`**
Modify `emit-rs` to check the registry *before* falling back to the `case` statement. Strip the explicit `level` and `hook-defun` parameters from recursive calls.

```lisp
(defun emit-rs (&key code (str nil) (level 0) (hook-defun nil))
  (let ((*emit-level* (if (zerop level) *emit-level* level))
        (*hook-defun* (or hook-defun *hook-defun*)))
    (if code
        (if (listp code)
            (let ((handler (gethash (car code) *emit-registry*)))
              (if handler
                  (funcall handler (cdr code))
                  ;; FALLBACK to the old CASE statement here temporarily
                  (case (car code)
                    ;; ... remaining unmigrated cases ...
                    (t (format nil "UNHANDLED: ~a" (car code))))))
            ;; Handle atoms (symbols, strings, numbers)
            (cond
              ((symbolp code) (substitute #\: #\- (format nil "~a" code)))
              ((stringp code) code)
              ((numberp code) (format nil "~a" code))))
        "")))
```

**Task 4.3: Iterative Migration**
Move operators one by one from the `case` statement in `core.lisp` to `def-emitter` blocks in `emitters.lisp`.

*Example in `emitters.lisp`:*
```lisp
(def-emitter + (args)
  (format nil "(~{~a~^+~})" (mapcar #'emit args)))

(def-emitter if (args)
  (destructuring-bind (condition true-statement &optional false-statement) args
    (with-output-to-string (s)
      (format s "if ~a ~a" (emit condition) (emit `(progn ,true-statement)))
      (when false-statement
        (format s " else ~a" (emit `(progn ,false-statement)))))))
```

> **CHECKPOINT 3:** After migrating `+`, `if`, `defun`, and `let`, run `sbcl --script tests.lisp`. Compare it to `baseline_output.txt`. It must match perfectly. Continue migrating until the `case` statement in `core.lisp` is completely empty and deleted.

## Phase 5: Externalizing Examples & `rustfmt` Integration

**Task 5.1: Delegate Formatting to `rustfmt`**
In `core.lisp`, update `write-source` to execute `rustfmt` via `uiop:run-program` (which uses the system `$PATH` rather than a hardcoded directory).

```lisp
(defun write-source (name code &optional (dir (user-homedir-pathname)) ignore-hash)
  (let* ((fn (merge-pathnames (format nil "~a" name) dir))
         (code-str (emit-rs :code code))
         (fn-hash (sxhash fn))
         (code-hash (sxhash code-str)))
    (multiple-value-bind (old-code-hash exists) (gethash fn-hash *file-hashes*)
      (when (or (not exists) ignore-hash (/= code-hash old-code-hash) (not (probe-file fn)))
        (setf (gethash fn-hash *file-hashes*) code-hash)
        (with-open-file (s fn :direction :output :if-exists :supersede :if-does-not-exist :create)
          (write-sequence code-str s))
        ;; Execute rustfmt
        (handler-case
            (uiop:run-program (list "rustfmt" (namestring fn)) 
                              :output *standard-output* 
                              :error-output *error-output*)
          (error (e)
            (warn "rustfmt failed. Code written unformatted. Error: ~a" e)))))))
```

## Phase 6: Final Compilation & Standalone Binary

Once the refactoring is complete and tested, you must optimize the compiler for speed.

**Task 6.1: Set Compiler Optimization Policies**
At the top of `core.lisp` (and optionally other files), change the development optimization declarations to production settings. This instructs SBCL to aggressively inline functions and skip safety checks for maximum AST processing speed.

```lisp
;; Production Optimization
(declaim (optimize (speed 3) (safety 1) (debug 0) (compilation-speed 0)))
```
Force a clean recompile: `(asdf:compile-system "cl-rust-generator" :force t)`

**Task 6.2: Dump a Native CLI Executable (Optional)**
If you wish to run the generator from the terminal without explicitly invoking SBCL scripts, dump a native binary.

In `core.lisp`, add an entry point:
```lisp
(defun main ()
  (let ((args (cdr sb-ext:*posix-argv*)))
    (if args
        (load (car args)) ; Load the user's AST lisp script
        (format t "Usage: rust-transpiler <file.lisp>~%")))
  (sb-ext:exit :code 0))
```

In your terminal, launch SBCL and execute the dump command:
```bash
sbcl
```
```lisp
(ql:quickload "cl-rust-generator")
(sb-ext:save-lisp-and-die "rust-transpiler" 
                          :toplevel #'cl-rust-generator::main 
                          :executable t 
                          :compression t)
```
You now have a standalone `./rust-transpiler` binary containing the fully optimized Lisp runtime and your refactored compiler.
